## Key Concepts

### Tree Traversals
- **Inorder (Left, Root, Right)**: Useful for BST to get sorted order
- **Preorder (Root, Left, Right)**: Useful for copying trees
- **Postorder (Left, Right, Root)**: Useful for deleting trees
- **Level Order (BFS)**: Useful for level-based problems

### Binary Search Tree Properties
- Left subtree contains only nodes with values less than the node's value
- Right subtree contains only nodes with values greater than the node's value
- Both left and right subtrees must also be binary search trees

### Common Patterns
1. **Recursive DFS**: Most tree problems can be solved recursively
2. **Iterative BFS**: Use queue for level-order traversal
3. **Divide and Conquer**: Solve for left and right subtrees, then combine
4. **Parent Pointer**: Track parent for path-related problems

## Complexity Reference

| Operation | Time Complexity | Space Complexity |
|-----------|-----------------|------------------|
| Traversal (DFS/BFS) | O(n) | O(h) / O(w) |
| Search (BST) | O(h) | O(h) |
| Insert (BST) | O(h) | O(h) |
| Delete (BST) | O(h) | O(h) |
| Height/Balance Check | O(n) | O(h) |

Where n = number of nodes, h = height of tree, w = maximum width of tree
